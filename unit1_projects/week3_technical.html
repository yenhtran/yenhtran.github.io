JavaScript

This blog will attempt to try to showcase the popularity of JavaScript in Web Development.

Why is JavaScript (Js) popular in web development?

Referenced from: Skillcrush

  1. Js is the only programming language that runs both in the frontend and the backend. Js also comes installed on every modern web browser vs. other languages where you have to install in your machine. Also, since Js is used on the frontend and runs al the interactive elements of a website, you see immediate return.

  2. Due to the introduction of AJAX, Js began to explode in popularity because AJAX allowed Js to work with data on the backend and allow for much more interations onwebsites. It is also used on mobile web devices.

  3. Job Security (article was written April, 2014)
    Due to explosion in popularity of Js, many employers are looking to hire people with Js skills. Just to list a few, 'JavaScript' is a keyword on most job posts on Indeed, jQuery (a popular Js library) is ranked #8 on the fastest growing keywords.

What are the great things about JavaScript?

Referenced from: Smashingmagazine

  1. Shortcut notations - Javascript used to be much more complicated to type out, but due to new shorcut notations, many objects and arrays have easy shortcuts that seem to be popular with web developers.

  2. JSON as a Data Format - Before the introduction of JSON, it sounds like getting content into a JavaScript-ready format was quite extensive. With the introduction of a JavaScript Object Notation (JSON), a lightweight data-interchange format, allowed developers to store complex data in a format that Js recognizes and can be used immediately.

  3. Native JavaScript Functions (Math, Array And String) - Prior to these functions being added to Js, people had to use a lot of looping and conditions. Due to these functions being added to Js, you can cut down a lot on these loops. In additona, a lot of other functions were added that makes like easier for a web developer.

  4. Event Delegation - Events make Web apps work. I'm not exactly sure what this does just yet, but it sounds like it makes products extensible without us needing to touch the core code.

  5. Anonymous Functions and The Module Pattern - In JavaScript, the variables usually don't have any scope, meaning that other scripts on the same page can access them and overide them. By using what is called 'The Module Pattern', allows one to build a variable's scope as needed by the user.

  6. Allowing for Configuration - You can add an object to your whole script called 'configuration'. In it, you store all the things people will likely change when using your script. Make the object public property so people can override it. You want to make it easy for people to use your code and alter it as needed.

What are it's limitations?

Referenced from: Daaq.net

  1. No graphic capabilities. While JavaScript can dynamically generate HTML, change document styles, and run simple animations, it can't draw things

  2. Can't read or write files. Js can't read or change the contents of your computer storage devices, but can request permission to do some form of file managment.

  3. It does not know how to use the network. While it knows how to request the Web browser to send and retrieve information from the network, it cannot engage in any network activity of its own.
